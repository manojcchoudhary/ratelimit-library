Quick Start

Get up and running in less than 2 minutes.

1. Add Dependency 

<dependency>
    <groupId>com.lycosoft</groupId>
    <artifactId>rl-adapter-spring</artifactId>
    <version>0.1.0-alpha</version>
</dependency>

2. Use the Annotation 

@RestController
public class OrderController {

    // Allow 100 requests per 60 seconds
    @RateLimit(requests = 100, window = 60)
    @GetMapping("/orders")
    public List<Order> getOrders() {
        return orderService.findAll();
    }
}

1. Add Dependency 

<dependency>
    <groupId>com.lycosoft</groupId>
    <artifactId>rl-adapter-quarkus</artifactId>
    <version>0.1.0-alpha</version>
</dependency>

2. Use the Annotation 

@Path("/orders")
public class OrderResource {

    // Allow 100 requests per 60 seconds
    @RateLimit(requests = 100, window = 60)
    @GET
    public List<Order> getOrders() {
        return orderService.findAll();
    }
}

Advanced Usage Patterns

Per-User Limits (SpEL) 

@RateLimit(
    key = "#user.id",
    requests = 100,
    window = 60
)
@PostMapping("/api/orders")
public Order createOrder(
    @AuthenticationPrincipal User user,
    @RequestBody OrderRequest request
) {
    return orderService.create(request);
}

Composite Keys 

@RateLimit(
    // Combine Tenant ID and User ID
    key = "#tenant.id + ':' + #user.id",
    requests = 50,
    window = 60
)
public void tenantAction() { ... }

Supports complex key generation logic directly in the annotation.

Tiered Limiting (Burst + Average)

Apply multiple limits to a single endpoint to handle bursts while maintaining long-term quotas. 

@RateLimits({
    @RateLimit(name = "burst", requests = 10, window = 1),       // 10 req/sec
    @RateLimit(name = "hourly", requests = 1000, window = 3600)  // 1000 req/hour
})
@GetMapping("/api/resource")
public Resource getResource() {
    return resourceService.fetch();
}

Configuration

Configure compiler modes and caching strategies via YAML. 

ratelimit:
  enabled: true
  spel:
    compiler-mode: IMMEDIATE
    cache-size: 1000

Prometheus Metrics

Monitor denial rates and throughput seamlessly. 

# Denial rate
rate(ratelimit_requests_total{result="denied"}[5m])

# Success rate
sum(rate(ratelimit_requests_total{result="allowed"}[5m]))
